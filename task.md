# er-diagram-chat 移植計画

## 概要

`er-diagram-chat`をLiamプロジェクトの`frontend/apps/app/app/(app)/app/(with-project-and-branch)/projects/[projectId]/ref/[branchOrCommit]/build/page.tsx`に移植する計画です。

## 現在の進捗状況

### 実装済みの機能

1. **コンポーネント構造**
   - BuildPageコンポーネントの基本構造を実装
   - ChatInterface、ThreadView、ChatNode、ChatInputNodeコンポーネントを実装
   - useERDChatフックを実装

2. **ERD機能**
   - ERDRendererコンポーネントを統合
   - カスタムノードタイプ（ChatNode、ChatInputNode）を定義

3. **チャット機能**
   - メッセージ表示と入力機能を実装
   - スレッド機能を実装
   - CSS Modulesを使用したスタイリングを実装

### 現在の課題

1. **依存関係の問題**
   - @xyflow/reactモジュールが見つからないエラー（インストールが必要）
   - 型定義の問題

2. **データ統合**
   - 現在はモックデータを使用しており、実際のAPIとの統合が必要

3. **カスタムノードタイプの統合**
   - ERDRendererとカスタムノードタイプの連携が未実装
   - 現在はERDRendererを直接使用しており、カスタムノードタイプを追加できていない

## ソースプロジェクト分析

`er-diagram-chat`は以下の機能を持つNext.jsアプリケーションです：
- ReactFlowを使用したERダイアグラムエディタ
- スレッド機能を持つチャットインターフェース
- ダイアグラム状態のバージョン管理

## 解決アプローチ

### ERDContentコンポーネントの拡張方法

1. **ERDContentコンポーネントのフォーク** ✅
   - @liam-hq/erd-coreパッケージのERDContentコンポーネントをコピーして修正
   - カスタムノードタイプを追加できるようにインターフェースを拡張
   - 欠点: パッケージの更新に追従する必要がある

2. **モンキーパッチング**
   - ReactFlowのnodeTypesを実行時に拡張
   - ReactFlowProviderのコンテキストにアクセスしてnodeTypesを修正
   - 欠点: 実装が複雑で脆弱になる可能性がある

3. **ERDContentコンポーネントの修正依頼**
   - @liam-hq/erd-coreパッケージの開発者にカスタムノードタイプを追加できるようにインターフェースの拡張を依頼
   - 欠点: 対応に時間がかかる可能性がある

4. **独自のReactFlowインスタンスを使用**
   - ERDRendererを使用せず、独自のReactFlowインスタンスを実装
   - ERDRendererと同様の機能を持つコンポーネントを作成
   - 欠点: 多くの機能を再実装する必要がある

### 依存関係の問題の解決方法

1. **@xyflow/reactのインストール**
   - @xyflow/reactパッケージをインストール
   - 型定義ファイルを追加

2. **型定義の作成**
   - 必要な型だけを定義した型定義ファイルを作成
   - 実際のパッケージをインストールせずに型エラーを解消

### ノード間の接続ロジックの実装方法

1. **エンティティノードとチャットノードの接続** ✅
   - エンティティノードからチャットノードへのエッジを作成
   - エッジのスタイルとアニメーションを設定
   - エッジのラベルを設定

2. **チャットノードの位置計算** ✅
   - エンティティノードの位置に基づいてチャットノードの位置を計算
   - 複数のチャットノードが重ならないように配置

3. **ドラッグ&ドロップの実装** ✅
   - チャットノードのドラッグ&ドロップを実装
   - ドラッグ中のエッジの更新を実装
   - ドロップ後の位置の保存を実装

### データ統合の実装方法

1. **APIクライアントの実装** ✅
   - ERDデータとチャットデータを取得・更新するAPIクライアントを実装
   - モックデータから実際のAPIへの移行を段階的に行う

2. **状態管理の統合** ✅
   - ERDの状態とチャットの状態を統合する状態管理を実装
   - バージョン管理機能を実装
